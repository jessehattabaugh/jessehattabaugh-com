<hx-fetch> and <hx-fetch-status> Documentation

Overview

<hx-fetch> and <hx-fetch-status> are custom web components designed to simplify handling fetch requests directly from HTML with enhanced user feedback through status indicators. <hx-fetch> captures form submissions and anchor clicks to perform asynchronous fetch requests, while <hx-fetch-status> provides visual feedback (loading, success, error) based on the fetch operation's state.

Features

  • <hx-fetch>: Enhances <form> and <a> elements to perform fetch requests without page reloads.
  • <hx-fetch-status>: Displays status indicators for fetch operations, allowing users to customize loading, success, and error messages.

Usage

<hx-fetch>

  1. Basic Setup: Add <hx-fetch> around a <form> or <a> element to capture its submission or click event. 
    <hx-fetch id="fetch1">
    <form action="/api/submit" method="post">
        <!-- form elements -->
    </form>
<hx-fetch>
  2. Attributes:
    • id: Unique identifier for the <hx-fetch> instance, used by <hx-fetch-status> to listen to specific fetch operations.

<hx-fetch-status>

  1. Integration: Place <hx-fetch-status> within your HTML and specify which <hx-fetch> instances it should respond to using the data-hx-targets attribute. 
    <hx-fetch-status data-hx-targets="fetch1">
    <div slot="loading">Loading...</div>
    <div slot="success">Success!</div>
    <div slot="error">An error occurred.</div>
<hx-fetch-status>
  2. Attributes:
    • data-hx-targets: Comma-separated list of id values from <hx-fetch> elements to monitor.
  3. Slots:
    • loading: Custom content to display during a fetch operation.
    • success: Custom content to display when a fetch operation completes successfully.
    • error: Custom content to display when a fetch operation fails.

Styling

<hx-fetch-status> uses Shadow DOM for encapsulation. Default styles hide all slots, showing only the relevant slot based on the fetch operation's state. Customize the visible slot's content through your HTML structure.

/* Inside your global stylesheet - affects slotted content */
hx-fetch-status::part(loading), hx-fetch-status::part(success), hx-fetch-status::part(error) {
    /* Styling for specific slots */
}

JavaScript Integration

<hx-fetch> automatically handles the fetch logic. Use standard JavaScript to listen for custom events if you need to execute additional logic or integrate with other components:

document.addEventListener('hx-fetch-success', (event) => {
    if (event.detail.id === 'fetch1') {
        console.log('Fetch 1 succeeded');
    }
});

document.addEventListener('hx-fetch-error', (event) => {
    if (event.detail.id === 'fetch1') {
        console.error('Fetch 1 failed');
    }
});

Best Practices

  • Unique IDs: Ensure id attributes on <hx-fetch> elements are unique to avoid unintended behavior.
  • Semantic Slots: Use semantic markup within <hx-fetch-status> slots to improve accessibility and SEO.
  • Error Handling: Implement robust error handling within fetch operations to provide clear user feedback.

Conclusion

<hx-fetch> and <hx-fetch-status> offer a declarative approach to handling fetch requests and displaying status feedback in web applications. By following the documentation and best practices, developers can enhance user experiences with minimal effort.
